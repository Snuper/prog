Сортировка по множеству ключей(индексация)
Пусть массив A - телефоный справочник, тогда его элементы-записи состоят из двух полей: фамилии и номера телефона.

TYPE subscriber = record name : string[20]; phone : longint; end; Var A : array [1..n] of subscriber; Пусть нам необходимо производить следующие операции:
1. По фамилии найти телефон(т.е. сортировка по фамилиям)
2. По телефону найти фамилию(т.е. сортировка по телефонам)

Для совмещения этих операций используется прием индексации.

A = 7 1 6 3 2 1 8 5 1 2 3 4 5 6 7 8 В массив B запишем номера элементов массива A в порядке возростания самих элементов

B = 2 6 5 4 8 3 1 7 - индексный массив Построение индексного массива:
1. В массив B записывается массив A
2. Производится любая сортировка, но при условии:
В операции сравнения - адресация через массив B
Перестановки только в B

B = (1,2,...,n) DO(i=1,..,n-1) DO(j=n,n-1,...,i+1) IF (a[b[j]]<a[b[j-1]]) b[j]<->b[j-1] FI OD OD DO(i=1,..,n) печать(a[b[i]]) Преимущества индексации:
  1. Возможность учитывать несколько индексов.

  2. Исключается копирование больших количеств инфориации.

  3. Возможность фильтрации данных - частный случай индексации.
