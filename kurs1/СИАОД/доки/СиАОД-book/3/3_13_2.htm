Двоичные Б - деревья
m=1. На странице 1-2 элемента.
При переходе к списочной структуре, страницы становятся условными. Характеристики страниц поддерживаются с помощью логических переменных. Поддержка листьев на одном уровне тоже с помощью логических переменных. В двоичном Б - дереве есть две высоты:

  1. аналог среднего времени поиска;
  2. высота дерева сравнив. по средней высоте.

Добавление вершины в двоичное дерево
VR - вертикальный рост(истина, если переходим на родительскую вершину)
HR - горизонтальный рост(истина, если добавление вершины происходит слева или справа)
Bal = 0, если только вертикальные ссылки
Bal = 1, если правая горизонтальная ссылка

Алгоритм добавления в ДБД

B2INSEART(D : данные; var p : pVertex) IF(p=NIL) New(p); p^.Data:=D; p^.Left:=NIL; p^.Right:=NIL; p^.bal:=0; VR:=true; ELSE IF (p^.Data>D) B2INSEART(D,p^.Left); IF (VR=true) IF(p^.bal=0) q:=p^.left; p^.left:=q^.right; q^.bal:=1; q^.right:=p; p:=q; VR:=false; HR:=true; ELSE p^.bal:=0; HR:=false; VR:=true; FI ELSE (HR=false) ELSE IF (p^.Data<D) B2INSEART(D,p^.right); IF(VR=true) VR:=false; HR:=true; p^.bal:=1; ELSE IF (HR=true) IF(p^.bal>0) q:=p^.right; p^.right:=q^.left; p^.bal:=0; q^.bal:=0; q^.left:=p; p:=q; VR:=true; HR:=false; ELSE HR:=false; FI FI FI FI FI FI АВЛ - дерево предпочтительнее, если добавлений меньше, а поиска больше. ДБД - наоборот.
